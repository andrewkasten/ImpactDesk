

# SWE Personal Project

## Two 3rd-Party REST API’s (4 pts)
   - 2 pts per API implementation
   - 1pt for using any endpoint from the API. 
   - 1pt for using additional API endpoints or API features such as search, pagination, or other query params/features available.
   
## Two CRUD Resources (4 pts)
   - 2 pts per CRUD resource implemented
   - 1 pt for Create and Read Endpoints
   - 1 pt for Update and Delete Endpoints 

## User Authentication (3 pts)
   - 1 pt for full-stack signup functionality - account creation
   - 1 pt for full-stack login - logging in with email or username and password, and receiving an auth token or equivalent.
   - 1 pt for having parts of your full-stack application that require authentication in order to be used, and logout functionality. Partial credit for these at the instructor’s discretion.

## Models & DB Design (3 pts)
   - 1 pt for having at least 2 Models, not counting the User Model, that has at least one field and is used in your application.
   - 1 pt for having relationships between your Models that are used as a part of your overall data model and application.
   - 1 pt at the instructor’s discretion - complexity of an individual model, of model validators/fields, of model relationships, of serializers, or of the data model as a whole, all would qualify.

## Styling, Presentation, & Navigation (3pts)
   - 1pt for having all of the following: frontend routing for your signup, login, home pages, having a Navbar, Sidebar, or similar UI feature
   - 2pts At instructor’s discretion for design, UI/UX, frontend look/feel, etc.  

## Feature Rich Application (3 pts)
   - If your frontend app is displaying to the user search results of a list of pokemon with data from PokeAPI, having frontend functionality for the user to be able to sort, filter, or paginate those search results.
   - Features specific to your project.
